# Math Quiz Platform — Deployment Guide

## One-Time Setup *(do this days before the event)*

---

### 1. Supabase

Follow **[SUPABASE_SETUP.md](./SUPABASE_SETUP.md)**, then:

```bash
python scripts/setup_supabase.py
```

All checks must show ✓ before continuing.

---

### 2. Questions

Prepare three JSON files (30 questions each):

```
questions/set_a.json
questions/set_b.json
questions/set_c.json
```

Format:
```json
[
  {
    "question_text": "What is 2 + 2?",
    "option_a": "3",
    "option_b": "4",
    "option_c": "5",
    "option_d": "6",
    "correct_answer": "B"
  }
]
```

Upload:
```bash
python scripts/upload_questions.py
```

Verify in Supabase: the `questions` table should have **90 rows** (30 per set).

---

### 3. Team Credentials

```bash
python scripts/generate_credentials.py
```

This creates `credentials.csv` — **keep this file private**.
Print and cut into individual slips, one per team.

Each slip contains:
- Team ID (e.g. `TEAM042`)
- PIN (e.g. `7291`)
- Quiz URL

---

### 4. Google Sheets

Follow **[GOOGLE_SETUP.md](./GOOGLE_SETUP.md)**, then:

```bash
python scripts/setup_google_sheets.py
```

---

### 5. GitHub + Render

1. Push code to a **private** GitHub repository
2. Go to [render.com](https://render.com) → **New Web Service** → Connect your repo
3. Render auto-detects `render.yaml` — review and confirm settings
4. Add **all environment variables** from `.env.example` in the Render dashboard:

| Variable | Where to get it |
|---|---|
| `SUPABASE_URL` | Supabase Settings → API |
| `SUPABASE_SERVICE_KEY` | Supabase Settings → API → service_role key |
| `JWT_SECRET` | Auto-generated by Render (`generateValue: true`) |
| `ADMIN_PASSWORD` | Choose a strong password |
| `GOOGLE_SHEET_ID` | From your Google Sheet URL |
| `GOOGLE_CREDS_JSON` | Entire service account JSON on one line |

5. Click **Deploy** → wait for build to complete (~3 min)
6. Verify:
   ```bash
   curl https://your-app.onrender.com/health
   # → {"status":"ok","timestamp":"..."}
   ```

---

### 6. Final Pre-Event Check

```bash
python scripts/pre_event_check.py --url https://your-app.onrender.com
```

**All 12 checks must show ✓ PASS** before event day.

---

## Event Day *(step by step)*

**Step 1** — 30 min before: run `pre_event_check.py` one more time.

**Step 2** — Arrive at venue, connect YOUR device to venue Wi-Fi.

**Step 3** — Open the admin dashboard on your phone or laptop:
```
https://your-app.onrender.com/admin-panel
```

**Step 4** — Login with `ADMIN_PASSWORD`.

**Step 5** — Click **"Lock Network"** → confirm the IP shown is the venue's public IP.
> If the IP is wrong, unlock and retry from a device on the correct network.

**Step 6** — Distribute credential slips to teams.

**Step 7** — Announce: "Open `https://your-app.onrender.com` and login with your Team ID and PIN."

**Step 8** — Wait until all teams show **"active"** status in the dashboard.

**Step 9** — Click **"Start Event"** → all 30-minute timers begin simultaneously.

**Step 10** — Walk the room with proctors. Watch the auto-refreshing dashboard for:
- 🔴 Red row = fullscreen exited (screen locked)
- 🟡 Orange row = tab switches > 3

**Step 11** — Red row: walk over, inspect. If innocent → click **"Unlock Screen"**.

**Step 12** — Orange row (tab switches): note for review.

**Step 13** — Re-login request: verify it's a genuine technical issue → click **"Approve Re-login"**.
The team's saved progress will be restored.

**Step 14** — At 30-minute mark: server auto-submits all remaining active teams.
Dashboard shows everyone as "submitted".

**Step 15** — Click **"End Event"** to officially close (catches any late submissions).

**Step 16** — Connect your laptop to a **mobile hotspot** *(not venue Wi-Fi)*.
> This is necessary because venue Wi-Fi is locked out from the server for security.

**Step 17** — Click **"Export to Google Sheets"** on the admin dashboard.

**Step 18** — Open your Google Sheet — final results are there with rank, score, and time taken.

## Troubleshooting

### App not loading for participants
**Fix**: Render free tier sleeps after inactivity. Upgrade to Starter tier ($7/mo) to prevent cold starts, or open the URL yourself 5 minutes before the event.

### "Network violation" shown even on venue Wi-Fi
**Fix**: The venue may use CGNAT (multiple users, one public IP). Click **"Unlock Network"** → have a participant reload and check their IP in Render logs → **"Lock Network"** again.

### Timer out of sync on participant screen
**Fix**: Timer resyncs automatically every 30 seconds. Tell the participant to stay on the page. The server timer is authoritative.

### Participant browser crashed, needs re-login
**Fix**: They click login again → if login_count > 0, a re-login request appears in your admin panel → click **"Approve Re-login"** → they log in again with the same credentials → progress is restored automatically.

### Export to Sheets fails
**Fix**: You are still on venue Wi-Fi (which is locked out). Switch your laptop to mobile hotspot, then export.

### Render app crashes under load (500 errors)
**Fix**: Scale up in Render → Settings → Scale. Free tier = 1 instance. Upgrade to handle 500 concurrent users.

### Score = 0 after submission  
**Fix**: Check the team's audit log in the admin dashboard. The most common cause is an empty `set_assigned` field — verify the team record in Supabase.

### Google Sheets export shows wrong email error
**Fix**: Share the Google Sheet with the service account's `client_email` address (found in `GOOGLE_CREDS_JSON`).

## CI/CD

Every push to `main` automatically:
1. Runs smoke tests (`pytest tests/`)
2. Triggers a Render deploy via webhook

**Setup**: In Render → Settings → Deploy Hooks → create a hook URL → save it as `RENDER_DEPLOY_HOOK_URL` in your GitHub repository secrets.
